# CLAUDE.md — AI 에이전트 작업 규칙

## 아키텍처 결정사항 (변경 금지)

### tmpdir는 반드시 /tmp/ 사용
- `mktemp -d /tmp/yolo-battle-XXXXXX` 형태를 유지할 것
- 프로젝트 내부(`.yolo-battle/` 등)에 작업 디렉토리를 만들지 말 것
- 이유: /tmp/는 OS가 자동으로 용량을 관리/삭제하므로 별도 정리 로직이 불필요함
- 프로젝트 디렉토리를 오염시키지 않고 .gitignore 관리도 필요 없음

## Git 브랜치 규칙 (필수)
- 반드시 자신의 브랜치에서 작업할 것 (예: `battle-coop-claude`)
- 다른 AI의 브랜치에 직접 push 금지
- 머지는 사용자의 지시가 있을 때만 수행

## 작업 시 주의사항
- 지시받은 작업만 수행할 것. 관련 없는 리팩토링이나 설계 문서 생성 금지
- 코드 변경 시 기존 동작을 깨뜨리지 않도록 주의
- 불필요한 디렉토리나 파일을 프로젝트 루트에 생성하지 말 것
